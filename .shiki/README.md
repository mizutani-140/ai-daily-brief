# .shiki Mirror

`.shiki/` is the repository-local mirror for Shiki state.

GitHub is the operational source of truth for Goals, Issues, Pull Requests,
Checks, Reviews, and merge evidence. This directory mirrors the same state in
portable files so an agent runtime can recover context without depending on
chat history.

`.shiki/manifest.json` is the canonical repository-local layout contract.
This README is the human-readable view of that manifest and is validated
against it.

## State Classes

Every manifest directory and file entry declares a `state_class`. `kind`
describes the artifact; `state_class` controls trust and PR mutation policy.

- `github-operational-source`: live GitHub state, authoritative and not stored
  under `.shiki`.
- `mirror`: repository-local mirrors of GitHub operational state.
- `append-only-evidence`: ledger evidence; PRs may append current-task entries
  but must not modify or delete existing entries.
- `governance-policy`: machine-readable governance config.
- `contract`: schemas, prompts, docs, and runtime contracts.
- `migration-state`: repository-local migration records.
- `workflow-runtime-evidence`: workflow-generated evidence such as
  `.shiki/gha`; must not be committed.
- `generated`: generated artifacts that must not be trusted as durable proof.
- `cache`: cache data that must not be committed or trusted.
- `local-only`: local runtime data that must not be committed.
- `template`: template files copied into targets.

## Runtime Evidence

Runtime evidence directories such as `.shiki/gha` are generated by workflow
runtimes and are not trusted PR evidence. They are intentionally excluded from
committed state.

Active lock files under `.shiki/locks` are mirror state created on demand and
are not required when no lock is active. Task lock intent is tracked in task
contracts.

## Empty Directories

Required tracked directories that may be empty are represented with `.gitkeep`.
Runtime-only directories do not require `.gitkeep`.

## Rule

If GitHub and `.shiki/` disagree, prefer GitHub operational state and create a
repair task to reconcile the mirror.
